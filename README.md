## Description

This project illustrates a robust event-driven microservices architecture built with NestJS and Kafka, designed for an e-commerce inventory and order management system. It is composed of three independent microservices, each with its own dedicated database:

Products Service – Manages product registration with fields such as name and price.

Storage Service – Handles inventory levels, allowing stock to be defined per product.

Orders Service – Manages customer orders, each referencing a product and including an address and a status field that transitions from PENDING to SHIPPED or DELIVERED.

Services communicate asynchronously via Kafka events, ensuring loose coupling and scalability. Data aggregation is performed by consuming and combining data from multiple services. For example, to return enriched order data with product details, the system fetches from both the Orders and Products services and serializes the combined data. Similarly, inventory reports merge information from the Storage and Products services.

Each microservice exposes a documented REST API using Swagger, which is accessible at the /api endpoint.

## Project setup

```bash
$ npm install

$ docker compose up -d
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```
